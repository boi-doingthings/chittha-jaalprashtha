DATA

Two primitives

- `torch.utils.data.DataLoader` --> Map-Style
- `torch.utils.data.Dataset` --> Iterable-Style
  • Map-Style : A map-style dataset is one that implements the getitem() and len() protocols, and represents a map from (possibly non-integral) indices/keys to data samples.
    • Useful for:
      • Read the data values at some i-th index in the dataset. Think like pandas dataframe indexing.
  • Iterable-Style: An iterable-style dataset is an instance of a subclass of IterableDataset that implements the iter() protocol, and represents an iterable over data samples.
    • Useful for:
      • Cases when random reads are expensive.
      • Batch size is data dependent.

Model Creation

In Pytorch, a model is a class that inherits from the nn.Module. There are two main components of model class:

  • A function __init__() : This stores the model components like: layers, dropouts etc
  • A function forward(): The forward flow of data in the model i.e how data flows from one layer to another.

from torch import nn
# Define Model

class NNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.flatten = nn.Flatten()
        self.relu = nn.ReLU()
        self.linear1 = nn.Linear(28*28,512)
        self.linear2 = nn.Linear(512,256*2)
        self.linear3 = nn.Linear(256*2,10)
        
    def forward(self,x):
        '''
        x : denotes the input data that will flow through the model
        '''
        x = self.flatten(x)
        x = self.relu(self.linear1(x))
        x = self.relu(self.linear2(x))
        return self.linear3(x)
    
model = NNet().to(device)
print(model)

def train(dataloader,model,loss_fn,optimizer):
    # You can directly apply the len function on dataloader.dataset to get the number of entries in the dataset.
    # Doing this on the dataloader returns the number of iteables formed
    size = len(dataloader.dataset) 
    model.train() #model in training mode
    for batch, (X,y) in enumerate(dataloader):
        X,y = X.to(device), y.to(device)
        
        # Compute prediction error
        pred = model(X)
        loss = loss_fn(pred, y)

        # Backpropagation
        optimizer.zero_grad() # clear the gradients and set to zero
        loss.backward() # do backprop
        optimizer.step() # udpate the gradients
        
        if batch % 50 == 0:
            loss, current = loss.item(), batch * len(X)
            print(f"loss: {loss:>7f}  [{current:>5d}/{size:>5d}]")
Code 
def test(dataloader, model, loss_fn):
    size = len(dataloader.dataset)
    num_batches = len(dataloader)
    model.eval() #model in eval mode so no gradient updates
    test_loss, correct = 0, 0
    with torch.no_grad():
        for X, y in dataloader:
            X, y = X.to(device), y.to(device)
            pred = model(X)
            test_loss += loss_fn(pred, y).item()
            correct += (pred.argmax(1) == y).type(torch.float).sum().item()
    test_loss /= num_batches
    correct /= size
    print(f"Test Error: \n Accuracy: {(100*correct):>0.1f}%, Avg loss: {test_loss:>8f} \n")
